Langkah skema utama Telegram -> Supabase
----------------------------------------
1. Jalankan SQL di SUPABASE_SCHEMA_NOTIFICATIONS.sql pada Supabase SQL Editor.
2. Deploy Supabase Edge Function telegram-to-supabase.
3. Simpan service role key sebagai Supabase Function secret:
   PANDUGIXVOLTKRAFT_SERVICE_ROLE_KEY
4. Simpan Telegram webhook secret sebagai Supabase Function secret:
   TELEGRAM_WEBHOOK_SECRET
5. Buat file lokal telegram_supabase_webhook_url.txt berisi URL function.
6. Buat file lokal telegram_webhook_secret.txt berisi secret yang sama dengan
   TELEGRAM_WEBHOOK_SECRET.
7. Jalankan:
   .\setup_telegram_supabase_webhook.ps1
8. Test dengan mengirim pesan ke bot/grup Telegram.

Status integrasi alarm
----------------------
Pada tahap ini Telegram sender sudah aktif dan sudah berhasil test kirim
pesan manual. Skema server langsung ke Supabase sudah ada sebagai cadangan,
tetapi tidak dilanjutkan karena skema utama yang dipilih adalah:

VoltKraft -> Telegram -> Supabase Edge Function -> Supabase table

Alarm otomatis dari DC/Annunciator/PQM/AVR/Kapasitor/AR belum disambungkan
ke Telegram/Supabase. Step berikutnya adalah deploy Edge Function, set secret
di Supabase, aktifkan webhook Telegram, lalu test dengan pesan Telegram.

Tempat menyimpan API key Supabase untuk skema utama
--------------------------------------------------
Service role key Supabase disimpan sebagai Supabase Edge Function secret:

PANDUGIXVOLTKRAFT_SERVICE_ROLE_KEY

Jangan simpan service_role key di frontend. Untuk skema utama ini, service_role
key juga tidak perlu disimpan di server VoltKraft.
